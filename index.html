Project idea:

Overview:

My project involves designing a chatbot app for new clients of a B&B or a hotel. My aim for this project is to provide the hotel/ B&B with an app that can answer minor questions with text or verbally regarding to the place they are staying. For example, the Chatbot app can tell where the broom is stored without the client requiring to contacting other people for assistance. The chatbot will also provide an option to speak with an employee if the chatbot was unable to answer the query. The Chatbot app can also provide the user with physical images to help supplement their queries. Returning to the previous example, the app will provide an image of the broom sitting in the cupboard.

Motivation:

While chatbots aren’t new to market, they have been rising in popularity for both clients and businesses. In a report created by Drift, 34-37% of participants (image 2, Drift (author and site), n.d., “Why are Chatbots Important?”, last accessed on 18/3/22) preferred to use a chatbot for getting detailed explanations, resolve conflicts or getting a quick answer in an emergency. Chatbots are important in reducing the stress on call centres with client queries. In a B&B/ Hotel setting, chatbots can be used to decrease the number of clients requiring assistance from human employees. Chatbots can provide clients with quick and direct assistance instead of waiting for a response from an employee.

Description:

The Chatbot app is designed to be simple and easy to use for the client but have complex background features. Each of the features are broken down below through explaining how the interface functions as a whole.

The main homepage will consist of 2 buttons: “Start chat” and “settings”. The setting button opens a menu for the user to change the visual look of the app or change text size, voice speed, etc. The “Start chat” initiates a conversation with the chatbot and a variety of start-up questions are shown based off what the chatbot can do (all are listed and described below).

About B&B/ hotel: Used to answer questions relating to the hotel such as the history and heritage of the place or general queries about the B&B/ hotels guidelines or facilities. This button can also be used to show the pricing and bundles for additional nights/ services the B&B/hotel provide.
Find something: Used to answer client queries regarding the location of household items that the B&B/ hotel provide. This can vary greatly from location of cooking utensils to the storage of extra linen. Items will have specific tags based off key words that can be linked to the client’s input through the search.
Local points of interest (POI): Used to guide clients to local businesses or locations that may be of interest to the client. This might be food venues for lunch or dinner, potential locations for a day out (e.g. beach) or local markets.
Report an issue: This button is used if there is a significant issue with the client’s venue. This button will branch off into several different problems: faulty/dangerous appliances, accidental damage or missing items.
None of these: Used to indicate that the client’s question is unrelated to the previous types or it is more specific. The chatbot will attempt will attempt to answer the question but will more likely hand over to an employee for an answer.

The client is able to input their question via a textbox at the bottom of the screen or record their question verbally and have the chatbot convert it to text. This is used to simplify user experience and allow the client to input the information via a method they prefer. The chatbot will also be able to respond in text or read the response out loud with assistance of text to speech.

There will also be a button on the bottom the screen for the user to report a bug. This is used by client to indicate to the hotel that there is specific flaw in the system. This is to ensure that the developers can keep improving the customer’s interaction with application.

As indicated above, if the chatbot is unable to provide an answer to a client they are redirected to a helpdesk where an employee will answer their query for the chatbot. This feature is implemented to ensure the client gets the information they need to receive even if the bot can’t provide it for them.

Tools and technologies:

The base app will require a physical server for the chatbot to run off. The server needs to store basic information of the B&B/ hotel to base its answers off. Within this project a free/ cheap server has to be sourced. A basic app builder software (such as MIT app inventor) will also be required with the creation of the app and its base functionality. Also, I would require chatbot building tools such as Google Dialogflow, Hubspot or Freshworks. The chatbot needs to be programmed to answer questions based off the answers stored in the server and be able to retrieve the correct information. The chatbot also must have the ability to contact an employee directly if it is unable to answer the client’s question. If this was implemented for a client, a firewall would also need to be installed to help protect the server from unauthorised access. Also, the client business will need to have phones and link them to the server to receive calls.

Skills required:

There is a variety of skills that needs to create to get the chatbot app working. Firstly, the app that the client interacts with the chatbot needs to be programmed. Also, the chatbot attached to the app needs to be programmed to provide answers to a variety of questions and learn from future client responses. Both of these skill requirements can be fulfilled by app developers with minor experience in creating these systems. A SQL server developer with some work experience is also required to create the server that will host all of the B&B’s/ Hotel’s information. Sourcing programmers with some experience with creating apps, servers and chatbots isn’t particularly difficult with enough funds. However, within the scope of this project these skills have to be self-taught.

Outcome:

If this chatbot project is successful it would improve both client and employee experience. Chatbots reduce the strain placed on employees to provide answers to clients of the B&B/hotel. The chatbot app will act as a middleman that answers simple questions for the clients about B&B/Hotel without contacting additional staff to get involved. The chatbot app will also reduce the time the client spends to get their answer since the chatbot automatically and instantly retrieves the answer the client will need. Also, the decreased demand for calls will reduce the pressure placed on employees to answer client questions. Hence employees can spend more time on the more complicated questions and deliver a better service for the clients.
